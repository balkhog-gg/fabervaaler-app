Trading Advisor

Prototype mono-fichier HTML pour structurer une décision de scalping à partir de captures M1 / M5 / M15 et de critères manuels. Ce n’est pas une IA magique : c’est un assistant de décision inspiré d’une logique order flow, zone mère, confirmation LTF, exécution et gestion du risque.

Cadre15m contexte • 5m zone • 1m exécution
Marché idéalNQ / MNQ puis ES / MES
DécisionLong / Short / No trade
ButRéduire l'impulsivité, pas promettre un edge automatique
Framework

Logique centrale

  1. Définir le contexte M15.
  2. Repérer la zone mère en M5.
  3. Attendre le retour du prix.
  4. Confirmer la rotation en M1.
  5. Entrer sur retest propre avec invalidation claire.
Filtre mental

Règles d'or

  • Pas de structure claire = pas de trade.
  • Pas de vraie zone = pas de trade.
  • Pas de confirmation M1 = prudence maximale.
  • Version agressive = taille plus petite.
  • Le meilleur trade est souvent celui qu'on laisse passer.

Inducement

Hiérarchie des zones

Toutes les zones ne se valent pas. Le prototype te force à distinguer une zone mère d'une zone secondaire qui attire les entrées trop tôt.

  • Zone extrême avant impulsion = priorité
  • Zone intermédiaire = méfiance
  • Balayage puis vraie réaction = scénario classique

GitHub

Déploiement simple

  1. Créer un dépôt GitHub nommé fabervaale-advisor.
  2. Ajouter ce fichier comme index.html.
  3. Pousser le repo puis activer GitHub Pages dans Settings → Pages.
  4. Sélectionner la branche main et le dossier root.
  5. Ton site sera en ligne en quelques instants.
